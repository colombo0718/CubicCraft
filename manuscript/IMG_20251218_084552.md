Code Craft 自定義 Blocks

遊戲中大量的功能是 Blockly 沒有的，須要自己定義屬於 Code Craft 的模塊。而要定義出一個有完整功能的 block，有兩項要素：

1. Blocks : 定義該 Block 的長樣。
2. Generator : 定義其所產生的程式碼。

兩者分別存放於 ccBlocks.js & ccGenerator.js，必須分開，否則會出錯。

於現階段，循序競速的模式下可能需要自定義的 Block 如下：

A. 控制相關
1. cpu-run : 置入處理器，每個瞬間要運算之內容。
2. switch engine : 單獨控制機體引擎之開關。
3. craft state : 系統取得機體狀態，如位置、速度等。
4. key press : 系統聽取玩家透過鍵盤給的指令。
5. print : 系統給玩家的提示，輔助字串。

B. 制 [?] 相關
6. time : 取得遊戲內度過時間。
7. check point : 取得每一個檢測點當下的位置資料。
8. progress : 目前目標在第 n 個。

2016.09.09 (六)

其中 cpu-run 是每個程式都要有的，而 3, 6, 7, 8 是負責系統對機體或環境的發報器，而 key press 則是對玩家的發報器。

如果一個系統沒有環境受器，又直接接收玩家的訊號，稱之為純手動系統。或完全關閉通路，玩家又能直接畫上的儀表板反視視覺化效果來作出反應。

反之，若一個系統完全無玩家直接控制，則為全自動系統。或稱完全的拉控 [?] 則為純自動系統。或稱完全的迴路，也許可以做為控制系統之演算方法的研究所用。

一般而言，完全的手動或自動都不見最佳方案，最好是混合 (hybrid) 的系統，以目前預設的方式向來是為自動導航，而按下 space 鍵轉為自動導航，兩者兼顧。
