Code Craft Blockly 形態轉換

從預設的程式 xml 字串，投放到 Blockly 工作區變成具體的 Blocks 結構，再進一步輸出成 javascript 程式碼，過程中經過了許多次的轉換。

Text
Blockly.Xml.textToDom ↓ ↑ Blockly.Xml.domToText
Dom
Blockly.Xml.domToWorkspace ↓ ↑ Blockly.Xml.workspaceToDom
Workspace
Blockly.Javascript.workspaceToCode ↓
Javascript

Text : Blockly 工作區內的程式不管有多龐大複雜，都可以用 xml 字串的方式作記錄，可做為儲存的途徑。
Dom : 視為中間的過渡帶。
Workspace : 程式的圖形化編輯區。
Javascript : 將 Blockly Workspace 中的內容以 javascript 的形式呈現。

2016.07.09 (六)

每當 Workspace 的內容有所更動，javascript 的程式碼就要即時回饋，呈現的方法是以 Ace web code editor 做為介面。要能正確運作，一樣有幾項要素：

1. 放置 Editor 的 Div 框
   `<div id="aceEditor" style="..."></div>`

2. 啟動功能的語法
   `var editor = ace.edit("aceEditor")`

其他還有基本設定：
`editor.getSession().setMode("ace/mode/javascript")`
`editor.setReadOnly(true)`

以及即時回饋的實現：
```javascript
workspace.addChangeListener(function() {
  text = Blockly.JavaScript.workspaceToCode(workspace)
  editor.setValue(text)
})
```
