Code Craft 嵌入 Blockly

作為 Code Craft 中唯一的編程環境，Blockly 嵌入後，必須有下列功能：

1. 遊戲進入編輯模式後，能顯示 Blockly 的圖形介面，否則隱藏。
2. 編輯完成進入遊戲後，機體須如實體樣式內容運作。
3. 有屬於 Code Craft 所特有的 Block，以感知或控制機體。
4. 每台機體有各自預設的程式，以 xml string 的方式存放，切換機體時取出。
5. 程式在編輯時，同步以 javascript 及 xml 輸出，也可載入先前寫好的程式的 xml 檔。

2016.07.08 (五)

要使 Blockly 的圖形介面正確顯示，必須要具備三項要素：

1. 放置 Blockly 的 Div 框
   `<div id="blocklyDiv" style="..."></div>`

2. 設定 toolbox 內容的 xml 標籤
   `<xml id="toolbox" style="display: none"> ... </xml>`

3. 啟動 Blockly 的 javascript 語法
   ```javascript
   var workspace = Blockly.inject('blocklyDiv', {
     toolbox: document.getElementById('toolbox')
   });
   ```

當然，最重要的是先引入 Blockly 相關的四個 javascript 檔：

1. `~/blockly.js`
   Blockly 的核心檔。
2. `~/blocks.js`
   官方預設的 Blocks 設定檔。
3. `~/javascript.js`
   blockly 轉 javascript 的產生器 generator。
4. `~/msg/js/en.js`
   文字設定檔，包含多國語言。
