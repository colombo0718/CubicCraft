Code Craft Blockly 程式運作

Blockly 程式的功能，就是要像真實世界的機器人控制程式一樣，能夠「時時刻刻」的取得機體當下的狀況以及環境的，並作出反應。

啟動自 Raffaello D'Andrea 於 TED 的演講。為此，Blockly 的程式中，必須有一個區塊是在每一個瞬間都執行一次，也就是會被 flash 呼叫到。另外還必須有全域變數的功能。當時間走到下一個瞬間，變數值不被改變，以達成動作維持 n 秒的效果。

程式結構如下：
```javascript
var blocklyCode = 
"var a = 0 ;" +
"var cpu_run = function() {" +
"  craft.switchEngine('left', true);" +
"};"

function triggerBlockly() {
  eval(blocklyCode)
  return {cpu_run : cpu_run}
}

function flash() {
  triggerBlockly().cpu_run()
}

runFlash = setInterval('flash()', 0)
```

2016.07.08 (五)

變數 blocklyCode 是由 Blockly 的內容轉換成 javascript 程式碼而來，當中必然包含函數 cpu-run，要將 js 的程式碼字串運作，要將字串放進 eval() 中。

而此時，如果不將 eval() 放進函數 triggerBlockly() 中，且在 flash() 中直接調用 cpu-run，如此一來大部分的功能可以運作，但若涉及全域變數，變數名稱可能會汙染到系統。

因此須將 eval() 放入 triggerBlockly() 以隔離變數，最後再將 cpu-run 以 return 的方式代出讓 flash() 調用。
