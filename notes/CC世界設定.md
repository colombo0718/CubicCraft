## 01｜世界觀

CC 宇宙（CodeCraft Universe）是一個**準物理模擬宇宙**——大量效法真實世界的仿真設定，也有為了遊戲性的虛構設定。

所有物體由一顆顆等大的礦石粒子構成。礦石經過鍛造結合後才能被人為控制、組合成機體。所有機體運動與環境互動，嚴格依照真實世界的剛體力學演算。

## 03｜故事劇情

玩家一開始扮演 **Leaf Lune 宇宙軍事學校**的新兵，與教官、學長、同學互動，逐步解鎖關卡、機體、環境與任務類型。

**畢業後**加入三個星際聯邦之一。三個聯邦各有盛產與稀缺礦物，衍生出各具風格的機體設計與戰鬥模式：
- 完成一般任務 → 獲得盛產礦物
- 完成掠奪型任務 → 獲得稀缺礦物
- 完成 PVP 任務 → 提升軍階 → 更高薪俸、更多機體容納空間

三個聯邦的資源結構差異，正是「策略蟹化」文化生成引擎的具體落地（見 `CC宇宙的風格與文化引擎.md`）。

## 04｜七種玩法方向

| 方向 | 數理需求 | 說明 |
|---|---|---|
| 機體操作 | 低 | 手動操控機體達成任務，以瞬間反應為主 |
| 程序員 | 高 | 編寫 Blockly/JS 程式讓機體自動控制 |
| 機體設計 | 高 | 構思並製作機體，針對不同任務微調 |
| 戰術分析 | 中 | 關卡難度評分、機體性能分析、戰後檢討 |
| 介面設計 | 中 | 設計人機操作介面與 HUD 儀表圖形 |
| 關卡設計 | 低 | 在既定遊戲模式下設計關卡供玩家下載 |
| 美術設計 | 無 | 機體塗裝設計，或作為獨立藝術創作素材 |

社群機制：程式、機體、賽道均可放到社群與其他玩家分享交流。

## 05｜五個開發階段

### 一、風馳電掣（The Cyclones）— 現階段
單剛體競速。每位玩家機體存在於獨立空間，其他玩家半透明呈現，無碰撞互動。
- 環境：輔助平面 + 閘門（可位移/轉動/縮放）
- 開放元件：推進器、重錘、鋼鐵、運算、發光、感測
- 推進器僅對剛體本身施力，不做反向噴氣施力
- 感測器此階段僅定義姿態計

### 二、烽火連天（The Flames）
加入熱兵器：20L 短槍、60L 長槍、罐頭飛彈。
- 子彈直接以終端速度射出，不考慮後座力
- 罐頭飛彈備有運算元件及追蹤器，可即時調整姿態鎖定目標

### 三、短兵相接（The Sword）
加入冷兵器。

### 四、千軍萬馬（The Troop）
開放多人連線。

### 五、風雲變色（The Mutation）
機體升級為多體系統——多關節機動戰士、艦長指揮整艘軍艦。

## 07｜CC 的教育定位

> CodeCraft 讓認知的建立不是透過數學與文字，而是在玩的過程中去感受虛擬太空環境中手動施予的控制與機體的反應，建立出粗略的認知與直覺之後，再從編寫控制程式的過程中回頭了解數學工具的使用。

學習路徑：
1. 手動飛行 → 感受推力與慣性
2. 寫 Blockly 程式 → 自動控制
3. 理解剛體力學 → 數學工具有了意義
4. 設計最優機體/控制策略 → 機器學習介入
